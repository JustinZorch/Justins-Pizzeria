The assignment - Pizzeria

  • Create an application with the described functionality by following the instructions below.
  • The purpose of this assignment is to gain insight into your way of programming, how you solve problems and which patterns you use.
  • The code does not have to be working, it is mainly the intention that you put down a concept.
  • Work carefully, this is more important than that all functionality ends up in your application.
  • Examples of the management and customer page can be found in these instructions.

Functionality

  1. Orders must be able to be placed on the customer page. The orders must be saved.
  2. The orders of the relevant pizzeria must be visible on the management page.
  3. The status should be changeable on the administration page. The current status must also be displayed on the customer page.
  4. At pizzeria "Dominos" you can only have delivery, pick up is not possible. At pizzeria "New York Pizza" you can only pick up, and there is no delivery. This must be processed in the ordering process.
  5. It should be easy to apply another storage method (text file, database, etc.).
  6. The customer indicates a preference for the way status updates are communicated (email, SMS, etc.). Status updates are communicated via customer preference. Which design pattern would you use for this and explain your choice.

Technical instructions

  • The attached code (/src) can be used as a starting point, but can be modified at your discretion.
  • Make sure that your code can be easily reused.
  • Focus your attention mainly on the business logic and less on, for example, configuring 3th party packages.

(Dutch) De opdracht - Pizzeria

  • Maak een applicatie met de beschreven functionaliteit aan de hand van onderstaande instructies.
  • Het doel van deze opdracht is om inzicht te krijgen in jouw manier van programmeren, hoe je problemen oplost en welke patterns je daarbij gebruikt.
  • De code hoeft niet werkend te zijn, het is vooral de bedoeling dat je een concept neerzet.
  • Werk zorgvuldig, dit is belangrijker dan dat alle functionaliteit in je applicatie belandt.
  • Voorbeelden van de beheer- en klantpagina vind je bij deze instructies.

Functionaliteit

  1. Op de klantpagina moeten bestellingen kunnen worden geplaatst. De bestellingen moeten worden opgeslagen.
  2. Op de beheerpagina moeten de bestellingen van de desbetreffende pizzeria zichtbaar zijn.
  3. Op de beheerpagina moet de status kunnen worden gewijzigd. Ook op de klantpagina moet de actuele status worden weergegeven.
  4. Bij pizzeria "Dominos" kun je alleen laten bezorgen, afhalen is niet mogelijk. Bij pizzeria "New York Pizza" kun je alleen afhalen, en wordt niet bezorgd. Dit moet in het bestelproces verwerkt worden.
  5. Er moet eenvoudig een andere storage methode toegepast kunnen worden (tekstbestand, database, etc.).
  6. De klant geeft een voorkeur op voor de manier waarop statusupdates gecommuniceerd worden (email, sms, etc.). Statusupdates worden via de voorkeur van de klant gecommuniceerd. Welk design pattern zou je hiervoor inzetten en licht je keuze toe.

Technische instructies

  • De bijgevoegde code (/src) kan als startpunt worden gebruikt, maar kan naar eigen inzicht worden gewijzigd.
  • Zorg ervoor dat je code eenvoudig herbruikt kan worden.
  • Richt je aandacht vooral op de business logica en minder op bijvoorbeeld het configureren van 3th party packages.
